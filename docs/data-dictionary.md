# AURUM — Data Dictionary

> Every field flowing through Kafka → Postgres landing → Snowflake RAW → SILVER → GOLD.
> Primary external source: SEC EDGAR XBRL APIs (`data.sec.gov`). Rate limit **10 req/s**; `User-Agent` header mandatory.

---

## Landing / RAW — `market_ohlcv_1m`

One row per ticker per minute bar, produced by the websocket market producer.

| Column | Type | Meaning |
|--------|------|---------|
| `ticker` | text | Stock symbol (e.g. `AAPL`) |
| `ts` | timestamptz | Bar timestamp (minute, UTC) |
| `open` / `high` / `low` / `close` | double | Prices for the minute bar |
| `volume` | bigint | Shares traded in the minute |
| `_ingested_at` | timestamptz | Consumer write time — incremental-load watermark |

Natural key: `(ticker, ts)`.

---

## Landing / RAW — `edgar_facts`

One row per reported value per metric per filing, from `companyfacts/CIK{cik}.json`.

| Column | Type | Source path | Meaning |
|--------|------|-------------|---------|
| `ticker` | text | `company_tickers.json` | Stock symbol |
| `cik` | text (10-digit, zero-padded) | EDGAR | **Central Index Key** — SEC's permanent company ID; builds all EDGAR URLs |
| `metric` | text | `us-gaap` / `dei` concept name | Which financial line item (see Metrics below) |
| `value` | double | `...units.{unit}[].val` | The number. **Raw dollars, not millions** — Apple revenue = `383285000000` |
| `period_end` | date | `...end` | Date the number is *as of* (balance-sheet snapshot date, or period end for income/cash-flow) |
| `filed_date` | date | `...filed` | When actually filed with SEC — can be weeks after `period_end`. **Use this for ML feature timestamps** (leakage guard) |
| `form_type` | text | `...form` | Filing type (see Form Types) |
| `accession_no` | text | `...accn` | Unique filing ID `XXXXXXXXXX-YY-NNNNNN` |
| `_ingested_at` | timestamptz | consumer | Watermark |

Natural key: `(cik, metric, period_end, form_type, accession_no)`.

|--------|------|---------|

## The 6 EDGAR Metrics AURUM Ingests

XBRL concepts pulled by the EDGAR producer:

### 1. `EarningsPerShareBasic`
- **Unit:** USD/shares. **Type:** duration (has start + end).
- Profit per share — net income / total shares. Most-watched earnings metric.
- Used for: P/E ratio.

### 2. `NetIncomeLoss`
- **Unit:** USD. **Type:** duration.
- The "bottom line" — revenue minus ALL costs. Negative = loss.
- Used for: `net_margin` = NetIncomeLoss / Revenues.

### 3. `Revenues`
- **Unit:** USD. **Type:** duration.
- Top line — total sales before costs.
- ⚠️ Some issuers report `RevenueFromContractWithCustomerExcludingAssessedTax` instead — producer falls back to that tag when `Revenues` is null.
- Used for: `net_margin`, `revenue_growth_qoq`.

### 4. `LongTermDebt`
- **Unit:** USD. **Type:** instant (point-in-time, no start date).
- Borrowings due > 1 year out. High debt + falling earnings = risk.
- Used for: `debt_to_market_cap`.

### 5. `EntityCommonStockSharesOutstanding`
- **Unit:** shares. **Type:** instant.
- ⚠️ Lives in the **`dei` namespace, not `us-gaap`** — pulled from `facts.dei.EntityCommonStockSharesOutstanding`.
- Used for: `market_cap` = shares × latest close.

### 6. `NetCashProvidedByUsedInOperatingActivities` (labeled `OperatingCashFlow`)
- **Unit:** USD. **Type:** duration.
- Real cash generated by operations — harder to manipulate than net income.
- ⚠️ AURUM's metric label is `OperatingCashFlow`; the actual XBRL concept name is `NetCashProvidedByUsedInOperatingActivities`. The producer maps label → concept.

## SILVER (Snowflake, dbt)

Cleaned, typed, deduplicated staging + intermediate models.

| Model | Grain | Content |
|-------|-------|---------|
| `stg_market_ohlcv` | ticker × minute | Typed bars, null/zero-volume filtered |
| `stg_edgar_facts` | cik × metric × period_end | Typed; filtered `form_type IN ('10-K','10-Q','8-K')`; **dedup: keep latest `filed_date` per (cik, metric, period_end)** — amendments supersede |
| `stg_news_sentiment` | item | Typed, deduped on natural key |
| `stg_companies` | ticker | Cleaned company/sector/CIK map |
| `int_stock_daily_enriched` | ticker × day | Minute bars rolled to daily OHLCV + technicals: `ma_30d`, `ma_90d`, `daily_return`, `vol_30d`, `sharpe_30d` |
| `int_engineered_financials` | ticker × quarter | Fundamental ratios (see formulas below) |

### Engineered financial formulas

| Column | Formula | Plain English |
|--------|---------|---------------|
| `price_to_earnings` | `latest_close / eps_basic` | Price paid per $1 of earnings. Typical range 10–30; high = growth expectation, low = value/cheap |
| `market_cap` | `latest_close × shares_outstanding` | Market's total valuation of the company |
| `debt_to_market_cap` | `long_term_debt / market_cap` | Leverage vs size. > 1.0 risky; < 0.3 low leverage |
| `net_margin` | `net_income / revenue` | Share of revenue kept as profit. 20%+ excellent; < 5% thin; negative = losing money |
| `revenue_growth_qoq` | `revenue / LAG(revenue) − 1` | Quarter-over-quarter growth; +0.10 = 10% |
| `ma_30d` / `ma_90d` | rolling mean of close | Price above MA = uptrend (short/long horizon) |
| `vol_30d` | stddev of daily returns, 30d window | Price volatility / risk |
| `sharpe_30d` | `(mean(daily_return) − rf_daily) / vol_30d × √252` | Risk-adjusted return; > 1.0 good, < 0 losing vs risk-free (rf = 4.5%/252) |

## GOLD (Snowflake, dbt)

| Model | Grain | Content |
|-------|-------|---------|
| `mart_features` | ticker × day | **ML feature store.** Daily technicals + latest-known fundamentals (as of `filed_date` ≤ day) + daily sentiment aggregates (mean intensity, pos/neg counts). Training extracts add the forward-return target |
| `mart_feature_summary` | ticker × day | Post-SHAP **selected** feature subset — the training view; regenerated by the feature-selection loop |
| `mart_stock_screener` | ticker (latest) | Analyst-facing joined mart — **the FastMCP query target.** Latest fundamentals + technicals + sentiment per ticker |

### `mart_stock_screener` columns

| Column | Source | Meaning |
|--------|--------|---------|
| `ticker`, `period_end` | edgar | Symbol, latest quarter |
| `eps_basic`, `revenue`, `net_income`, `long_term_debt`, `shares_outstanding` | edgar | Raw fundamentals (raw dollars) |
| `price_to_earnings`, `market_cap`, `debt_to_market_cap`, `net_margin`, `revenue_growth_qoq` | engineered | Ratios (formulas above) |
| `ma_30d`, `ma_90d`, `vol_30d`, `sharpe_30d` | technicals | Trend + risk indicators |
| `latest_price`, `price_date` | market | Most recent close + its date |
| `sentiment_7d`, `news_count_7d` | news | Mean sentiment intensity + item count, trailing 7 days |

## Form Types

| Form | Frequency | Content |
|------|-----------|---------|
| `10-K` | Annual | Full-year audited financials — most complete/trustworthy |
| `10-Q` | Quarterly (×3) | Unaudited quarterlies; Q4 lives inside the 10-K, no separate 10-Q |
| `8-K` | Event-driven | Material events: earnings releases, CEO changes, M&A. Often carries preliminary EPS before the 10-Q |
| `10-K/A`, `10-Q/A` | Rare | Amendments — supersede by latest `filed_date` |

## Gotchas

| Gotcha | Fix |
|--------|-----|
| Same metric, same `period_end`, multiple rows | Keep latest `filed_date` (amendment) |
| `Revenues` null for some issuers | Fallback: `RevenueFromContractWithCustomerExcludingAssessedTax` |
| Values are raw dollars | Don't rescale on ingest; format at presentation |
| Instant vs duration metrics | Balance-sheet items (`LongTermDebt`, shares) have no start date |
| 10-K = full year, 10-Q = one quarter | Never mix form types in growth calcs |
| Shares outstanding in `dei` namespace | Pull from `facts.dei.…`, not `us-gaap` |
| SEC 10 req/s limit | 0.1 s sleep between companyfacts calls — do not remove |
| Fundamentals leak into the past | Feature-join on `filed_date`, never `period_end` |

## Key External Endpoints

| Endpoint | Purpose |
|----------|---------|
| `https://www.sec.gov/files/company_tickers.json` | ticker → CIK for all public companies |
| `https://data.sec.gov/submissions/CIK{cik}.json` | Full filing history, one company |
| `https://data.sec.gov/api/xbrl/companyfacts/CIK{cik}.json` | All facts ever reported — main EDGAR source |
| `https://data.sec.gov/api/xbrl/companyconcept/CIK{cik}/us-gaap/{concept}.json` | One metric, one company (lighter) |
| `https://www.sec.gov/Archives/edgar/daily-index/{Y}/QTR{N}/master.{YYYYMMDD}.idx` | Daily new-filings index — incremental trigger (see `edgar-incremental-ingestion.md`) |
| Wikipedia *List of S&P 500 companies* | Ticker universe + sector + CIK |
